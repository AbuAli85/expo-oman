================================================================================
PROBLEM 3: WRONG TO FIELD FORMAT
================================================================================

SYMPTOM:
--------
- Execution appears
- Error: "Missing value of required parameter 'to'"
- OR: "The 'to' field must be a 'string'"

CHECK:
------
1. Open Module 3 or 4 (Email module)
2. Check "To" field
3. What format is it?

FIX:
----
Should be EXACTLY this format:
```json
[{"value": "{{1.1}}"}]
```

NOT these:
- `["{{1.1}}"]` ❌
- `["{{1.`1`}}"]` ❌
- `{{1.1}}` ❌

HOW TO FIX IN MAKE.COM:
------------------------
1. Open Module 3 or 4
2. Click on "To" field
3. Delete current value
4. Click "Add item"
5. Select "Object"
6. Add: `{"value": "{{1.1}}"}`
7. Save

================================================================================
PROBLEM 4: BACKTICKS IN VARIABLES
================================================================================

SYMPTOM:
--------
- Variables show as literal text in email
- Name doesn't appear: "Dear {{1.`0`}},"
- Email address shows as "{{1.`1`}}"

CHECK:
------
1. Open Module 3 or 4
2. Check all fields for backticks
3. Look for: `{{1.`1`}}`, `{{1.`3`}}`, `{{1.`0`}}`

FIX:
----
Remove ALL backticks:

In Filter:
- `{{1.`3`}}` → `{{1.3}}`

In To field:
- `["{{1.`1`}}"]` → `[{"value": "{{1.1}}"}]`

In Subject:
- `{{1.`0`}}` → `{{1.0}}`

In HTML:
- `{{1.`0`}}` → `{{1.0}}`

HOW TO FIX:
-----------
1. Open Module 3 or 4
2. Go through each field
3. Find and replace:
   - Find: `{{1.`3`}}` → Replace: `{{1.3}}`
   - Find: `{{1.`1`}}` → Replace: `{{1.1}}`
   - Find: `{{1.`0`}}` → Replace: `{{1.0}}`
4. Save
